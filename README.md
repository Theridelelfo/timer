# 🛠 DevSecOps Learning Path — Timer Web App

## 🎯 Obiettivo

Questo repository non si concentra sulla complessità del codice applicativo (un timer minimale in JavaScript nativo), ma funge da PoC (Proof of Concept) per dimostrare come progettare e implementare un ciclo di vita del software moderno e sicuro.

L'obiettivo è mostrare come integrare una pipeline DevSecOps su un micro-frontend, ponendo l'attenzione su processo, qualità e automazione più che sulla logica applicativa.

Attraverso questo progetto è possibile acquisire:
- Mentalità DevOps
- Integrazione della sicurezza (DevSecOps)
- Automazione CI/CD
- Qualità del codice e testing

## 🧠 Principi guida

### 1. Shift Left
Qualità e sicurezza vengono introdotte il prima possibile nel ciclo di sviluppo, non alla fine.

### 2. Automazione
Tutto ciò che può essere automatizzato viene automatizzato per garantire coerenza e ridurre errori manuali.

### 3. Fail Fast
Gli errori devono emergere il prima possibile (una build fallita è preferibile a un bug in produzione).

### 4. Everything as Code
Configurazioni, pipeline e regole fanno parte del codice e vengono versionate insieme al progetto.

## 🚀 Metodo di lavoro

Il lavoro segue un ciclo iterativo:

1. Analisi dello step
2. Implementazione guidata
3. Verifica (lint + test)
4. Feedback e gestione dei dubbi
5. Iterazione

## 📊 Obiettivo finale

Al termine del percorso avrai costruito:

✅ Una pipeline CI/CD completa e automatizzata  
✅ Un sistema di testing affidabile  
✅ Controlli di sicurezza integrati nel ciclo di sviluppo  
✅ Un meccanismo di deploy automatico  
✅ Un’architettura del codice pulita e modulare  

# 🧠 Risultato reale

Non avrai semplicemente costruito un timer.

Avrai acquisito una comprensione pratica di:
- come lavora un team di sviluppo moderno
- come si struttura e si mantiene un progetto professionale
- come integrare sicurezza e sviluppo in un unico flusso

---

👉 Ogni step deve essere **compreso**, non solo eseguito.
