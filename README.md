# 🛠 DevSecOps Learning Path — Timer Web App

## 🎯 Obiettivo

Questo repository non si concentra sulla complessità del codice applicativo (un timer minimale in JavaScript nativo), ma funge da PoC (Proof of Concept) per la progettazione e l'implementazione di un ciclo di vita del software sicuro. L'architettura è pensata per dimostrare l'integrazione di una pipeline DevSecOps industriale su un micro-frontend.

L'obiettivo è acquisire:
- Mentalità DevOps
- Integrazione della sicurezza (DevSecOps)
- Automazione CI/CD
- Qualità del codice e testing

## 🧠 Principi guida

### 1. Shift Left
La qualità e la sicurezza vengono introdotte **prima**, non alla fine.

### 2. Automazione
Tutto ciò che può essere automatizzato **deve essere automatizzato**.

### 3. Fail Fast
Gli errori devono emergere subito (build fallita > bug in produzione).

### 4. Everything as Code
Configurazioni, pipeline e regole vivono nel repository.


# 🚀 Metodo di lavoro

Procederemo così:

1. Analisi step
2. Implementazione guidata
3. Feedback e dubbi
4. Iterazione
